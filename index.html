POST /v2/survey/{id}/lock

Places a lock on a survey so that other users cannot edit it. The user requesting edit will have the lock. If the survey already has an editor, the editor will receive an email notifying that a user would like access. Only team members or guests with write permissions can request survey edit.

Parameters

Name In Type Required Description
id path integer true Survey ID

GET /v2/site/{site_id}/surveys

Surveys that belong to folders will not be included in the response, unless a search is being made. Results are ordered by name ascending by default if no sorting is specified.

Parameters

Name In Type Required Description
siteId path integer true Site ID
q query string false Search surveys and folders
status query integer false Return only surveys in a specific status (0 for archived surveys or 1 for open surveys).
page query integer false Page to retrieve from the entire result set.
page_size query integer false Max amount of objects to return per page in the response. Defaults to 100.
sort_by query string false Sort by column names (comma separated). Use - prefix on column name to sort descending.
external_folder_id query string false Filter surveys by folder using string of external_id (UUID).
